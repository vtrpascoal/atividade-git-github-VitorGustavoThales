Requisitos Funcionais 

* RF01 - Cadastro de alunos
O sistema deve permitir cadastrar alunos com informações básicas, como nome, identificação, data de nascimento e sexo.

* RF02 - Cadastro de funcionários
O sistema deve permitir o cadastro de funcionários, incluindo professores, coordenadores e administradores.

* RF03 - Cadastro de disciplinas
O sistema deve permitir cadastrar disciplinas e vinculá-las aos cursos disponíveis. 

* RF04 - Matrícula de alunos em disciplinas
O sistema deve permitir inscrever alunos em disciplinas específicas.

* RF05 - Registro de notas
O sistema deve permitir registrar as notas dos alunos em cada disciplina.

* RF06 - Consulta de alunos cadastrados
O sistema deve exibir a lista de alunos cadastrados.

* RF07 - Exibição de alunos por disciplina
O sistema deve mostrar quais alunos estão matriculados em cada disciplina.

* RF08 - Verificação de aprovação ou reprovação
O sistema deve calcular e informar se o aluno foi aprovado ou reprovado com base nas notas cadastradas.

* RF09 - Geração de relatórios
O sistema deve gerar relatórios de alunos, funcionários e disciplinas cadastradas.

* RF10 - Armazenamento temporário de dados
O sistema deve salvar os dados inseridos durante a sessão utilizando armazenamento local do navegador.






Requisitos Não Funcionais 

* RNF01 - Interface intuitiva
O sistema deve possuir uma interface simples e de fácil utilização.

* RNF02 - Tempo de resposta rápido
O sistema deve responder às ações do usuário em até 2 segundos.

* RNF03 - Compatibilidade com navegadores modernos
O sistema deve funcionar corretamente nos principais navegadores atuais.

* RNF04 - Persistência durante a sessão
Os dados devem permanecer acessíveis enquanto a sessão do navegador estiver ativa.

* RNF05 - Organização modular do código
O código deve ser estruturado em módulos/classes para facilitar manutenção.

* RNF06 - Facilidade de manutenção
O sistema deve permitir alterações e correções futuras sem impactar outras funcionalidades.

* RNF07 - Validação de dados
O sistema deve validar os dados inseridos antes de armazená-los.

* RNF08 - Segurança básica de entrada
O sistema deve impedir entradas inválidas ou vazias nos campos obrigatórios.

* RNF09 - Escalabilidade estrutural
A estrutura deve permitir expansão futura com novas funcionalidades.

* RNF10 - Portabilidade
O sistema deve poder ser executado em diferentes sistemas operacionais por meio do navegador.
